Google Maps for ExpressionEngine

Written By Justin Kimbrell

Overview

These examples focus on the new features added as of v2.2. The biggest changes to the plugin are the new methods: exp:gmap:center, exp:gmap:radius, and exp:gmap:marker method does all the same things as exp:gmap, except it doesn't initialize another map. Instead it reu-uses one that already been initialized noted by the id attribute. Also note, exp:gmap tag will work the same.

Example #10 - Using the Google Map fieldtype

{exp:gmap:init id="map10" class="map" gmap_field="map_gmap"}{title}{/exp:gmap:init}

The most notable addition to v2.2 is the Googl Map fieldtype. Once you have installed and added the fieldtype to a channel, and added coordinates, this example shows how to plot them on a map inside a template.

Just like plotting coordinates or addresses from custom fields in a channel, you can plot coordinates added with the Google Map fieldtype by using the gmap_field="name_of_custom_field" attribute.

Example #11 - Add an array of points using a string

{exp:gmap:init id="map11" class="map" lat_lng="(25, 45)(-45, 12)(10, -86)"}{title}{/exp:gmap:init}

You can also add a string of coordinates to the map by using the lat_lng="(latitude, longitude)" attribute.

Note, the lat_lng attribute uses the same format the Google Map fieldtype saves to the channel. If the string is not formatted properly, there will likely be errors.

Example #12 - Make use of the additional methods

{exp:gmap:init id="map12" class="map"}Marker 1{/exp:gmap:init}

{exp:gmap:marker id="map12" latitude="0" longitude="0"}Marker 1{/exp:gmap:marker} {exp:gmap:marker id="map12" class="map" lat_lng="(50, 0)"}Marker 2{/exp:gmap:marker}
{exp:gmap:marker id="map12" latitude="50" longitude="10"}Marker 3{/exp:gmap:marker}
{exp:gmap:marker id="map12" class="map" lat_lng="(40, 0)(40, 10)"}Marker 4{/exp:gmap:marker}
{exp:gmap:marker id="map12" class="map" lat_lng="(30, 0)"}Marker 5{/exp:gmap:marker}
{exp:gmap:marker id="map12" class="map" lat_lng="(20, 0)"}Marker 6{/exp:gmap:marker}
{exp:gmap:marker id="map12" class="map" lat_lng="(10, 0)"}Marker 7{/exp:gmap:marker}

You can now easily add markers to the map outside exp:gmap method. This method can be used inside another plugin that loops (like Geocoder). All of the same tags supported in the exp:gmap will work in the exp:gmap:add_marker method.

Note, every marker that is added to the map will extend the bounds and cause the map automatically adjust the center and zoom to display all the markers. If you would like to add a point and NOT extend the bounds (change the center or zoom) of the map, use the property extend_bounds="false".

Example #13 - Making more use the additional methods

{exp:gmap:init id="map13" class="map"}Marker 1{/exp:gmap:init}

{exp:gmap:marker id="map13" gmap_field="map_gmap" extend_bounds="no" icon="/-/media/blue-marker.png"}{title}{/exp:gmap:marker}
{exp:gmap:center id="map13" address="Africa"}
{exp:gmap:zoom id="map13" zoom="2"}

By default, the plugin extends the bounds of the map to automatically adjust zoom and center to fit all the markers into view. There are a number of way to override this option. The easiest is by defining a zoom or zoom_field parameter. However, maps with multiple points, this won't work all the time. There are two new methods, exp:gmap:zoom, that specifically address this scenario (among many others too). So in this case, add the extend_bounds="no" attribute to exp:gmap:marker method.

Always use the exp:gmap:center or exp:gmap:zoom method. Once the map is initialized, you may set the center and zoom separately.

Note, the exp:gmap:center tag to get the same effect.
Ex, {exp:gmap:center id="map13" address="Africa" zoom="2"}

The extend_bounds is needed because Google Maps extends the bounds asynchronously, which is what overrides the standard map.setZoom() method. (As in each methods gets called procedurally, but when the bounds request calls back to the map, it overrides the zoom that was just set.) There are other ways to do this, but given the scope of the ExpressionEngine tags, I see this as the best way. I'm welcome to other thoughts and ideas if there is a better way, but this way isn't too bad. In a nutshell, if your zoom isn't behaving how you thought, make sure you add extend_bounds="no" to every method that adds markers to the map.

Example #14 - Show one info window at a time, or not...

{exp:gmap:init id="map14" class="map"}

{exp:gmap:marker id="map14" gmap_field="map_gmap" show_one_window}{title}{/exp:gmap:marker}
{exp:gmap:marker id="map14" latitude="0" longitude="-20" icon="/-/media/blue-marker.png"}This is the blue marker.{/exp:gmap:marker}

You can easily restrict the map from showing more than one info window at a time by adding show_one_window="true".

You can also change the icon by define a by defining a file path for example, icon="/some/file/path/icon.png"

Example #15 - Add custom styles and markers

{exp:gmap:init id="map15" style_link="/-/script/gmap-styles.js" class="map"}Marker 1{/exp:gmap:init}

{gmap:marker id="map15" gmap_field="map_gmap" icon="/-/media/blue-marker.png"}{title}{/exp:gmap:marker}

You can also define a file path to an external JavaScript file that contains a Google Map style JSON object. You can use this Google Maps styling wizard to create the JSON object. Simply save it to the desired location, and link to it using style_link="/some/file/path/file.js".

By default the plugin looks for an object name stylez, which can be changed to anything you like by define style_object="someObjectName".

Note, the style_link and style_object properties must be located in the exp:gmap or exp:gmap:init methods, the exp:gmap:marker.

Example #16 - Auto-populated drop down of markers

{exp:gmap:init}

{exp:gmap:marker id="map1" class="map" channel="map_demo" latitude_field="map_latitude" longitude_field="map_longitude"}{title}{/exp:gmap:marker}

{exp:gmap:marker id="map1" class="map" channel="map_demo" gmap_field="map_gmap" url_title="random-places-not-in-us"}{title}{/exp:gmap:marker}

{exp:gmap:dropdown id="map1" class="some-desired-class"}

This is an example of how you can use the exp:gmap:dropdown method to create a drop down that gets automatically populated by the markers that are added to the map. The same as the other methods, append the id attribute to the tag that refers to your map.

By default, the titles of the entries are used as the values of the drop down. However, if there is no title, then the plugin refers to the data that is used to populate the info windows. If you wish to use info window contents even when a title is present, use the attribute add_title_to_dropdown="no".
