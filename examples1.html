Overview

So a client asks you, "Can't you just make a Google Map that I can plot my own points by entering the address' in the admin panel?" Alternatively, they just want their business location plotted on a map within a specific page on the site. Whether it's a map with single point on a contact page, or an array of points from a database, the process is relatively the same. This feature is requested on almost every site I have put together for a client.

Google Maps takes much of the pain away, but the platform is so robust, it can be overwhelming to implement basic features without spending an hour (or more) of your time. (And it required the knowledge of JavaScript.) This plugin tries to make the process even easier by providing the easiest Google Map implementation possible, but at the same time providing an extremely diverse and powerful way to build maps using complex dynamic datasets.

This plugin combines the entire exp:channel:entries tag and all the parameters with all the properties used to initialize a Google Map. The end result with save you loads of time and create virtually any Google Map you desire.

Example #1 - Geocode a single address and add it to the map

{exp:gmap:init id="map1" class="map" address="Indianapolis, IN"}Indianapolis, IN{/exp:gmap:init}

This is the most basic implementation of the plugin. Just add an address and give the map an id. We also added a class so we could tell the Map to be 400 x 250 and float right within the stylesheet, this attribute is not required.

Additionally, you can create an info window (for every location) by adding some inner content to the tag. You may enter HTML and channel variables, relationships, and conditional statements. Anything supported within the exp:channel:entries will work in this plugin.

Note, if you plan on making info window's you must close all the tags, even if ther are intended don't have them. This is an issue with the EE parsing engine, not the plugin.

The id is required and must only contain alphabetical and numerical characters with the exception of an underscore. Any other characters will result in a JavaScript error.

Example #2 - Define a custom zoom

{exp:gmap:init id="map1" class="map" address="Indianapolis, IN" zoom="12"}Indianapolis, IN{/exp:gmap:init}

By taking the previous example a little further, let's adjust the zoom to get a better feel for the location. By default, the zoom and center point are calculated automatically which can be overridden by appending the zoom attribute with a numerical value (1-20).

You also change the center point by adding a center attribute with an address as the value. In this case, we left the pin point as the center.

Example #3 - Geocode multiple address' from a string

{exp:gmap:init id="map3" class="map" address="Indianapolis|Maine|California"}Indianapolis, IN{/exp:gmap:init}

Again, iterating on the last example, you can add more than one address to the address field by separating them with a | character.

Example #4 - Using Google Map's API parameters

{exp:gmap:init id="map4" class="map" latitude="30.27" longitude="-86" zoom="12" maxZoom="15" minZoom="10" scrollwheel="false" draggable="false"}Indianapolis, IN{/exp:gmap:init}

This example demonstrates how you would incorporate different Google Map options to change the interaction of the map. You can add any number of these properties to customize your map. In this example we removed the scrollwheel zooming features, limited the minimum and maximum zoom, and set the default zoom to 12.

Example #5 - Using the channel:entries tag to geocode

{exp:gmap:init id="map5" class="map" channel="map_demo" address_field="map_address" city_field="map_city" state_field="map_state" zipcode_field="map_zipcode" country_field="map_country"}{/exp:gmap:init}

This example illustrates how to plot points on a map using ExpressionEngine's channel data. When retrieving data from channels, you must define the fields that contain the address data. There are 5 fields thats can be used to make up a the full address, but you can use only one field if desired, depending on your database schema. (For instance, you may store the entire address in a single field, or break it out into multiple fields. It's up to you.)

The values of these field(s) should be the same and the corresponding custom field name. (They are concatenated in the following order: address_field, city_field, state_field, zipcode_field, and country_field.)

Note, this tag is still producing dynamic results based on the URL just as the exp:channel:entries tag. You can disable this function by adding dynamic="no". For more information, refer to the Channel Reference.

Do not use these fields for latitude and longitude, only for geocoding an address stored within your channel data.

Example #6 - Using the channel:entries tag to geocode

{exp:gmap:init id="map6" class="map" channel="map_demo" address_field="title" dynamic="no" url_title="golden-gate-bridge-san-francisco-california-94129"}<p>This is some sample markup, anything could go here. You can even include the channel variables, relationships, conditional states, and anything the channel tag supports.</p> Address: <b>{title}</b>{/exp:gmap:init}

This example shows how to use the exp:channel:entries parameters to display a specific result. This should be familiar to you if you used the exp:channel:entries tag before, and this example is just to iterate the exact implementation. Again, every parameter in the exp:channel:entries tag is available to use.

In this example, the dynamic property is set to no and the url_title is hardcoded to display a single result. Anything could be possible here, but the point is the plugin is not limiting.

Example #7 - Using channel:entries tag with coordinates

{exp:gmap:init id="map7" class="map" channel="map_demo" latitude_field="map_latitude" longitude_field="map_longitude"}{map_latitude}, {map_longitude}{/exp:gmap:init}

This example demonstrates the ability to add dynamic latitude and longitude variables, which is a feature new to v2.1. If you are trying to display are large number of points, you will need to use latitude and longitude to get around Google's geocoding query limit, or have access to the premium API.

Notice the tag is looping through many more records than being displayed on the map. You can have a channel with both types of points, those that need geocoding, and those that don't. If the longitude or latitude field is blank, the record is skipped. You can even include both latitude and longitude fields in the same tag as the address fields.

Example #8 - Using channel:entries tag with coordinates

{exp:gmap:init id="map8" class="map" channel="map_demo" latitude_field="map_latitude" longitude_field="map_longitude" zoom_field="map_zoom" url_title="random-place-2" }{map_latitude}, {map_longitude}{/exp:gmap:init}

Iterating on the last example, this map implements the zoom_field parameter to define a custom field that contains a zoom level. This field is only intended to be used with maps with a single entry. If more than one location is displayed, the last record will set the final zoom.

Example #9 - Using channel:entries tag with coordinates

{exp:gmap:init id="map9" class="map" channel="map_demo" latitude_field="map_latitude" longitude_field="map_longitude" zoom_field="map_zoom"}{map_latitude}, {map_longitude}{/exp:gmap:init}

This example shows what happens when you have multiple locations on the map and still use the zoom_field parameter.
